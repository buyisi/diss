# 农药库存管理系统

一个零成本/极低成本的农药库存管理系统，支持手机浏览器扫码入库、出库，实时查询库存和过期预警。

## 功能特点

- **入库管理**：支持扫码输入条码，自动识别商品信息，记录批次号、数量、生产日期和过期日期
- **出库管理**：扫码确认商品，输入批次号和出库数量，自动扣减库存
- **库存查询**：实时查看所有商品库存情况
- **过期预警**：自动检测30天内即将过期的商品
- **手机扫码**：集成ZXing-js库，支持手机浏览器调用摄像头扫码
- **响应式设计**：适配电脑和手机端，可添加到手机主屏幕获得接近APP的体验

## 技术栈

- **前端**：HTML5 + CSS3 + JavaScript + EJS模板
- **后端**：Node.js + Express
- **数据库**：SQLite（本地）/ Supabase（云端）
- **扫码库**：ZXing-js

## 本地运行

1. **安装依赖**
   ```bash
   npm install
   ```

2. **启动服务器**
   ```bash
   npm start
   ```

3. **访问系统**
   打开浏览器，输入 `http://localhost:3000`

## 云端部署（零成本方案）

### 1. 数据库迁移到Supabase

1. 注册Supabase账号：https://supabase.com/
2. 创建新项目，获取数据库连接信息
3. 修改 `db.js` 文件，使用Supabase SDK连接数据库

### 2. 部署到Vercel

1. 注册Vercel账号：https://vercel.com/
2. 连接GitHub仓库
3. 配置环境变量（数据库连接信息）
4. 自动部署

### 3. 部署到Netlify

1. 注册Netlify账号：https://www.netlify.com/
2. 连接GitHub仓库
3. 配置环境变量
4. 自动部署

## 手机使用指南

1. **访问系统**：在手机浏览器中输入部署后的URL
2. **添加到主屏幕**：
   - iOS：点击分享按钮 → 添加到主屏幕
   - Android：点击菜单按钮 → 添加到主屏幕
3. **扫码操作**：
   - 进入入库或出库页面
   - 点击"手机扫码"按钮
   - 允许摄像头权限
   - 对准条码进行扫描

## 注意事项

- **本地运行**：数据存储在本地SQLite数据库中，无需网络连接
- **云端部署**：需要稳定的网络连接
- **扫码性能**：手机摄像头质量和光线条件会影响扫码速度
- **免费额度**：云服务的免费额度有流量/存储限制，满足小仓库日常使用
